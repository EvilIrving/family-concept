# Views 目录 Agent 规范

本目录只放可组合的页面级 SwiftUI 视图，用于承载 tab 根页面、业务页面和少量页面级编排。

这里的文件负责组织信息架构、页面布局、sheet 呈现、导航入口和页面内交互流转，不负责沉淀可复用视觉原子。

可跨页面复用的按钮、卡片、行容器等基础视图应放到 `../UI/Components/`，toast、banner 等反馈机制应放到 `../UI/Feedback/`。

页面允许依赖 Store、Model、Components 和 Feedback，但不要在本目录内定义通用样式常量、重复组件或独立反馈控件。

表单录入优先通过 sheet 承载，单字段修改优先内联完成，页面设计应服从 tab 根层级不超过 2 层的导航约束。

新增页面文件名使用 PascalCase，并与页面主类型一致；预览、页面私有拆分和轻量页面状态可与主页面同文件维护。

菜单页的“新增菜品”属于高频关键动作，入口默认使用右下角 `FloatButton`，不再把主入口放在页面头部常规按钮区。

“新增菜品”sheet 默认半高呈现，打开后立即聚焦“菜名”输入框并弹出键盘，顶部操作区固定提供取消与保存，键盘弹起时保存动作仍需可见可点。

表单结构固定为“菜名 + 分类 + 食材”，其中分类优先使用常见分类的快捷胶囊选择，并保留自定义输入兜底，不应默认让用户从空白文本开始思考分类。

食材录入优先实现为 tag 式快速输入，用户输入后通过回车、空格或顿号即时落成标签并可单个删除；不要要求用户长期依赖“用顿号分隔”的纯文本格式。

保存校验只阻止缺少关键字段的提交，错误反馈使用字段下方轻提示或 toast，不使用阻塞弹窗；保存成功后优先清空表单、关闭 sheet，并给出即时反馈。
