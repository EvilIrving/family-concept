# Components 目录 Agent 规范

本目录只放跨页面复用的基础 UI 组件与轻量组合组件，例如按钮、卡片、行容器、字段壳层和通用视觉结构。

组件应尽量保持无业务语义、低耦合和可预测，优先通过参数表达状态与样式，不在组件内部持有复杂业务流程。

如果某个视图已经绑定具体页面路径、页面文案或厨房业务语义，应留在 `../../Views/`，而不是下沉到这里伪装成通用组件。

反馈性质的控件，例如 toast、banner、skeleton 和错误提示，不放在本目录，应归入 `../Feedback/`。

组件实现优先复用主题色、圆角、间距和层级规则，保持绿色系视觉一致性，避免在单个组件里定义偏离主题的独立视觉体系。

文件命名使用 PascalCase，并与主类型一致；只有当多个页面明确复用且语义稳定时，才新增独立组件文件。
