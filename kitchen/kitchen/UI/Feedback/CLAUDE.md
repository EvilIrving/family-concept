# Feedback 目录 Agent 规范

本目录只放界面反馈能力，包括 toast、inline banner、空态提示、加载占位和错误状态等即时反馈组件。

反馈组件的职责是表达结果、状态和可恢复动作，必须轻量、非阻塞，并保持与 SwiftUI 默认过渡动画一致的连续体验。

删除、撤销等破坏性流程应优先提供 undo toast 或局部反馈，不在这里实现确认弹窗导向的阻塞流程。

本目录不放业务页面、不放通用布局容器，也不承载领域状态计算；反馈文案可以接收业务数据，但业务决策应留在 View 或 Store。

若某个反馈样式会在多个页面复用，应优先在这里抽象，再由 `../../Views/` 调用；若只是视觉原子而非反馈语义，应放到 `../Components/`。

文件命名使用 PascalCase，并以反馈能力或反馈容器命名，确保从名称即可判断其触发方式与展示语义。
