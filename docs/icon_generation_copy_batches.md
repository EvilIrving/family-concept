# 图形生成批次复制稿

每批仅需复制下方 text 框内全文。**说明：多数扩散生图并不会输出真 PNG Alpha，却会误把「透明」画成灰白棋盘格**当作示意。正文已改为：**显式禁止棋盘格**，并写明若无法生成透明底则改用**单色键控背景（文中 #FF00FF 品红）**以便在 Figma/Photoshop/Affinity 等中一键抠除，再上屏前导出带 Alpha。

## 图标语义原则（生成时每项必读）

UI 小图标在 **16–24pt** 的用途与 **SF Symbol** 同级：**一眼读出动作或状态**，不要靠「画面内容丰富」取胜。

- **单一主符号**：每项尽量只有一个图形母题（加号、减号、叉号、放大镜、对勾、时钟、箭头等）。品牌气质靠 **圆角、配色、轻量描边/填充**，不靠堆物件。
- **禁止隐喻叠层**：不要把「动作」再叠在餐盘、小票、蒸汽、菜篮、小屋等装饰上（例如：**加量 = 纯加号**，不要叠盘子；**搜索 = 纯放大镜**，不要叠碗）。
- **移除/删除类**：购物车移除、取消等 → **柔和的 xmark（叉）** 或极简 **废纸篓轮廓**；避免「盘子 + 叉 + 高光」式的徽章插画。
- **少层次、弱体积**：避免复杂 3D、多层高光、细小编排不下的细节；**扁平 / 略厚矢量** 即可。
- **空状态插画（第 8–11 批）**：允许比工具栏图标稍叙事，但**每个画面主物件不超过两件**；同样避免「放大镜+盘+卡+碎片」式装满。

下列清单为 `kitchen` 内语义、状态与代码视图的对照表，与下文第 1–26 批生成主题对应（**第 21–26 批**为源代码审计补充分拆：偏好与裁切流程、历史订单 UI、占位与 Tab、点餐条等）。**各批 `text` 框内具体条目须遵守本节。**

**复制时**：请整段复制每个批次内从「一次生成」到三条编号为止的全文（含通用说明）；若你自行改写某条，仍以文首原则为准，避免把「厨房氛围」理解成「多叠几个物件」。
