## 一、App 信息（App Information）

路径：App Store Connect → 食单 → 综合 → App 信息

### 1.1 内容版权

点击「设置内容版权信息」，选择：

**你的 App 或 App 中的任何部分是否包含、显示或访问第三方内容？**

→ 选「**否**」

除非你的菜品图片素材引用了他人的版权作品，否则选否。

---

### 1.2 年龄分级

点击「设置年龄分级」，按以下回答：

| 问题类别 | 选择 |
|----------|------|
| 不受限的网页访问 | **否** |
| 用户生成内容、信息和聊天 | **否**（菜品图片和菜单内容不构成用户间通信）|
| 广告 | **否** |
| 低俗笑话 | **否** |
| 恐怖/惊悚题材 | **否** |
| 使用或提及烟、酒或毒品 | **否** |
| 医疗或治疗信息 | **否** |
| 健康或保健主题 | **否**（美食佳饮类，非健康建议）|
| 成人或暗示性主题 | **否** |
| 色情内容或裸体 | **否** |
| 卡通或幻想暴力 | **否** |
| 现实暴力 | **否** |
| 枪支或其他武器 | **否** |
| 赌博 | **否** |
| 模拟赌博 | **否** |
| 竞赛、抽奖箱 | **否** |

**预期结果：4+**

---

### 1.3 App 加密文稿

你的 App 使用 HTTPS（TLS），属于 Apple 操作系统内置的标准加密，**无需**上传加密合规文件。

若 App Store Connect 要求声明，在 Info.plist 中已包含：

```
<key>ITSAppUsesNonExemptEncryption</key>
<false/>
```

若出现「你的 App 是否使用加密？」问题 → 选「**否，仅使用操作系统内置加密**」。

---

### 1.4 数字服务法（DSA）

如果 App 在欧盟地区上架，需声明是否为交易者（Trader）：

- 个人开发者 → 选「是」，填写联系信息
- 暂不发布欧盟地区 → 可跳过

选择不发布欧盟地区时，此项留空即可。

---

### 1.5 中国大陆 ICP 备案号

如果 App 在中国大陆 App Store 上架，需要填写 ICP 备案号。备案流程：

1. 在阿里云/腾讯云提交 ICP 备案
2. 获得 ICP 备案号后填入此处
3. 首次提审可先留空，审核时会提示补全

## 二、App 隐私（App Privacy）

路径：App Store Connect → 食单 → 综合 → App 隐私

### 2.1 隐私政策 URL

已填写：`https://evilirving.github.io/family-concept`

---

### 2.2 隐私问卷

点击「开始」后，逐项按以下填写。

#### 第一步：是否收集数据？

**选：是**

#### 第二步：数据类型勾选

只勾选以下 4 类，其他全部不勾选：

| 数据类型 | 勾选 | 子类 |
|----------|------|------|
| 联系信息 | ✅ | 其他用户联系信息 |
| 标识符 | ✅ | 用户 ID |
| 用户内容 | ✅ | 照片或视频、其他用户内容 |
| 购买 | ✅ | 购买历史 |

**不勾选**：健康与健身、财务信息、位置、敏感信息、联系人、浏览历史、搜索历史、使用数据、诊断数据、设备 ID、其他数据。

#### 第三步：逐项详答

**联系信息 → 其他用户联系信息**

| 问题 | 回答 |
|------|------|
| 是否关联至用户身份？ | **是** |
| 是否用于追踪？ | **否** |
| 用于何种目的？ | ✅ App 功能 |

> 说明：用户注册时提供的用户名和昵称，通过 HTTPS 传输至服务端，用于账户管理、订单协作和厨房成员展示。密码不在客户端持久化。

**标识符 → 用户 ID**

| 问题 | 回答 |
|------|------|
| 是否关联至用户身份？ | **是** |
| 是否用于追踪？ | **否** |
| 用于何种目的？ | ✅ App 功能 |

> 说明：服务端为每个账户签发的唯一账号 ID，存储于本地 UserDefaults（key: `accountID`），用于维持登录会话和标识操作人。

**用户内容 → 照片或视频**

| 问题 | 回答 |
|------|------|
| 是否关联至用户身份？ | **是** |
| 是否用于追踪？ | **否** |
| 用于何种目的？ | ✅ App 功能 |

> 说明：用户通过相机拍摄或相册选取菜品照片，上传至服务端作为菜品封面，展示给同一厨房的成员。照片上传至 Cloudflare R2，临时文件在上传后清理。

**用户内容 → 其他用户内容**

| 问题 | 回答 |
|------|------|
| 是否关联至用户身份？ | **是** |
| 是否用于追踪？ | **否** |
| 用于何种目的？ | ✅ App 功能 |

> 说明：用户创建的菜品名称、食材、分类、厨房名称和订单内容，存储于服务端供同一厨房成员共享协作。

**购买 → 购买历史**

| 问题 | 回答 |
|------|------|
| 是否关联至用户身份？ | **是** |
| 是否用于追踪？ | **否** |
| 用于何种目的？ | ✅ App 功能 |

> 说明：App 内购买（50 道菜品额度、无限菜品额度）完成后，将 StoreKit 签发的 JWS 交易收据发送至服务端验证以解锁权益。支付卡信息由 Apple 直接处理，App 和后端均不接触。

#### 第四步：是否用于追踪？

**选：否**

理由：不使用广告网络、不共享数据给数据经纪商、不将用户数据关联至第三方 app 或网站的广告行为。

## 四、提交前核对表

在点击「提交审核」前确认：

- [ ] 内容版权已设置
- [ ] 年龄分级问卷已完成（4+）
- [ ] 加密声明已处理
- [ ] 隐私政策 URL 可正常访问
- [ ] 隐私问卷 4 类数据已逐项填写完成
- [ ] ICP 备案号（如需）已填写或确认留空
- [ ] 元数据（名称、副标题、关键词）已定稿
- [ ] 截图已上传（5.5" iPhone 至少 3 张）
- [ ] 审核备注已编写（包含测试账号和测试步骤）
- [ ] 构建版本（Build）已选择
