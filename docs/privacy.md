# Cooklist 隐私政策

本隐私政策适用于 Cooklist iOS 应用及其相关服务。

## 我们收集的信息

当你注册、登录或使用应用功能时，我们会处理以下信息：

账号信息，包括用户名、昵称、账号 ID 与登录会话令牌。

你主动提交的内容，包括厨房名称、邀请码、菜品名称、食材、订单内容，以及你上传的菜品图片。

设备本地偏好设置，包括主题模式、是否完成引导页、最近一次进入的厨房 ID，以及用于保持登录状态的本地会话信息。

## 我们如何使用这些信息

我们仅将上述信息用于提供应用核心功能，包括账号登录、恢复登录状态、创建或加入厨房、管理成员、展示菜单、上传和显示菜品图片、创建和处理订单，以及保存你的界面偏好。

我们不会将你的个人信息用于广告投放，也不会将你的信息出售给第三方。

## 图片与用户内容

你上传的菜品图片会被上传到应用使用的远程存储服务，用于在应用内展示菜品封面图。

你提交的厨房名称、菜品信息、订单内容和其他用户内容会发送到应用服务器，以实现多人协作和业务功能。

## 第三方服务

根据当前版本代码，应用未集成统计、广告归因或第三方追踪 SDK。

应用会通过系统网络框架与应用自有后端服务通信，并使用远程图片存储地址加载你已上传的菜品图片。

应用集成了开源图片加载库 Nuke，用于在设备本地缓存菜品图片以减少重复下载。该库只在本地运行，不会向任何第三方服务器发送你的数据。

## 追踪与定向广告

我们不进行跨应用或跨网站追踪。

我们不使用广告标识符，也不会基于第三方数据进行定向广告投放。

## 本地存储

应用会在你的设备本地存储部分信息，例如登录会话令牌、账号 ID、昵称、最近一次进入的厨房 ID、引导页状态和主题模式，以便维持登录状态和改善使用体验。

为减少网络流量和提升加载速度，应用还会把已加载过的菜品图片缓存在设备的应用容器目录内。缓存仅在本地使用，会在容量超限或系统清理时自动失效；卸载应用时会一并删除。

## 数据共享

我们仅在提供应用服务所必需的范围内处理和存储你的数据。

如果你所在的厨房包含其他成员，你提交的厨房成员信息、菜品信息、订单内容和相关图片会在该厨房的成员之间显示。

除提供服务所必需外，我们不会主动向无关第三方披露你的个人信息，除非法律法规要求。

## 你的选择

你可以选择不上传图片，但某些需要图片的功能可能因此无法完成。

你也可以通过退出登录来清除设备上的登录状态；部分服务器侧业务数据仍可能按照服务运行需要保留。

## 未成年人

如果你是未成年人，请在监护人同意和指导下使用本应用。

## 隐私政策更新

如果本隐私政策发生变更，我们会通过应用内页面或更新后的公开链接提供最新版本。

## 联系我们

如果你对本隐私政策或个人信息处理方式有疑问，应通过应用公开提供的联系渠道与我们联系。
